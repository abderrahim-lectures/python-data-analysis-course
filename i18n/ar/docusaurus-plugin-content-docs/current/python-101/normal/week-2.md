# الأسبوع 2: التحكم في تدفق التنفيذ

<span className="gamified-flourish">🔀 هذا الأسبوع، تتوقف برامجك عن تنفيذ نفس الأسطر الثلاثة في كل مرة وتبدأ باتخاذ القرارات.</span>

## 🎯 أهداف التعلّم

بنهاية هذا الأسبوع ستكون قادرًا على:
- تفريع سلوك البرنامج باستخدام `if` / `elif` / `else`.
- دمج الشروط باستخدام `and`، `or`، و`not`.
- تكرار العمل باستخدام حلقتي `while` و`for`، واختيار المناسبة منهما.
- استخدام `range()` للتكرار عددًا ثابتًا من المرات، واستخدام `break`/`continue` للتحكم بالحلقة مبكرًا.
- تضمين الحلقات والشروط داخل بعضها لحل مسائل بها أكثر من عنصر متغيّر.

## الدرس

### التفريع: `if` / `elif` / `else`

جملة `if` هي تعريف مجزّأ (piecewise). قارن (القيم بين علامتي التنصيص هنا بالإنجليزية فقط لأن محرّك عرض الرياضيات KaTeX المستخدم في الموقع لا يدعم رسم الأحرف العربية داخل الصيغ الرياضية؛ negative تعني "سالب"، zero تعني "صفر"، positive تعني "موجب"):

$$
f(x) = \begin{cases} \text{"negative"} & x < 0 \\ \text{"zero"} & x = 0 \\ \text{"positive"} & x > 0 \end{cases}
$$

```python
x = -3
if x < 0:
    print("negative")
elif x == 0:
    print("zero")
else:
    print("positive")
```

المسافة البادئة هنا ليست مجرد أسلوب — إنها **الصياغة نفسها**. كل ما هو مُزاح تحت `if` ينتمي لذلك الفرع؛ لا تملك بايثون `{ }` لتحديد الكتل. تحتاج الكتلة إلى جملة واحدة على الأقل — فإن أردت فرعًا لا يفعل شيئًا عمدًا (مثلًا أثناء رسم هيكل البرنامج قبل ملئه)، استخدم `pass`، وهي جملة مهمتها الوحيدة ألا تفعل شيئًا:

```python
if x < 0:
    pass   # TODO: handle negative numbers later
else:
    print("non-negative")
```

### دمج الشروط: `and`، `or`، `not`

تمامًا مثل $\land$، $\lor$، $\neg$ في المنطق، تمتلك بايثون `and`، `or`، `not` لدمج التعبيرات المنطقية:

```python
age = 20
has_ticket = True

if age >= 18 and has_ticket:
    print("Welcome in")

if age < 13 or age > 65:
    print("Discount applies")

if not has_ticket:
    print("Buy a ticket first")
```

تتبع نفس جداول الصدق التي تتوقعها: `and` تكون صحيحة فقط عندما يكون *كلا* الطرفين صحيحين؛ `or` تكون صحيحة عندما يكون *طرف واحد على الأقل* صحيحًا؛ `not` تعكس القيمة المنطقية. كما تقوم بايثون بـ"التقييم المختصر" (short-circuit): في `a and b`، إذا كانت `a` بالفعل `False`، فإن بايثون لا تُقيّم `b` إطلاقًا، لأن التعبير كله يجب أن يكون `False` بغض النظر. يهم هذا عندما يكون تقييم `b` مكلفًا أو غير آمن — فمثلًا `x != 0 and 10 / x > 1` لا تخاطر أبدًا بالقسمة على صفر، لأنه إذا كانت `x != 0` تساوي `False`، تُتخطى القسمة تمامًا.

### `while`: التكرار حتى يفشل الشرط

حلقة `while` هي أقرب شيء لعلاقة تكرارية رياضية مع شرط توقف:

```python
n = 5
while n > 0:
    print(n)
    n = n - 1
print("liftoff!")
```

يطبع هذا `5 4 3 2 1 liftoff!`. تعيد الحلقة فحص `n > 0` قبل كل تكرار — إن نسيت تحديث `n` داخل الحلقة، فلن تصبح `False` أبدًا، وستحصل على حلقة لا نهائية.

`while` هي الأداة الصحيحة عندما لا تعرف مسبقًا عدد التكرارات التي ستحتاجها — يعتمد شرط التوقف على شيء يحدث *أثناء* الحلقة، مثل إدخال المستخدم أو بحث قد ينتهي مبكرًا:

```python
guess = None
target = 42
while guess != target:
    guess = int(input("Guess the number: "))
print("Correct!")
```

### `for`: التكرار عبر متتالية معروفة

تسير حلقة `for` مباشرة عبر متتالية من القيم — فكّر فيها كـ $\sum_{i \in S}$ حيث $S$ هي أيًا ما تكرر عليه، إلا أنك تنفّذ كودًا لكل $i$ بدلًا من جمع أرقام:

```python
for letter in "cat":
    print(letter)   # c, a, t — واحد في كل سطر
```

تُنتج `range(n)` المتتالية $0, 1, \dots, n-1$ — بالضبط مجموعة الفهارس التي تستخدمها لـ $\sum_{i=0}^{n-1}$:

```python
total = 0
for i in range(5):
    total = total + i
print(total)   # 0+1+2+3+4 = 10
```

تعمّم `range(start, stop)` و`range(start, stop, step)` هذه الفكرة، مما يعكس متتالية حسابية $a, a+d, a+2d, \dots$:

```python
for i in range(2, 10, 2):
    print(i)   # 2, 4, 6, 8 — تبدأ من 2، تتوقف قبل 10، بخطوة 2
```

الخطوة السالبة تُنقص: `range(5, 0, -1)` تُنتج `5, 4, 3, 2, 1`، نفس المتتالية التي بناها مثال `while` الخاص بالإقلاع في الأسبوع 1 يدويًا.

### `break` و`continue`

تُنهي `break` الحلقة فورًا؛ تنتقل `continue` إلى التكرار التالي دون إنهاء التكرار الحالي:

```python
for i in range(10):
    if i == 5:
        break        # توقف تمامًا عندما تصل i إلى 5
    if i % 2 == 0:
        continue     # تخطَّ طباعة الأعداد الزوجية
    print(i)          # يطبع 1, 3
```

### التضمين: حلقات وشروط داخل بعضها

يمكن لجسم حلقة — أو فرع `if` — أن يحتوي على حلقة أو `if` أخرى، تمامًا كما يمكن لحالات دالة معرّفة بالتجزئة أن تتضمن شروطًا إضافية. كل مستوى تضمين يضيف مستوى إضافيًا من المسافة البادئة:

```python
for row in range(1, 4):
    for col in range(1, 4):
        print(row * col, end=" ")   # end=" " تطبع مسافة بدلًا من سطر جديد
    print()                          # انتقل لسطر جديد بعد كل صف
```

يطبع هذا جدول ضرب صغير بحجم 3×3:

```
1 2 3
2 4 6
3 6 9
```

تُنفَّذ الحلقة الداخلية بالكامل في *كل* تكرار للحلقة الخارجية — لثلاثة تكرارات خارجية وثلاثة داخلية لكل منها، هذا $3 \times 3 = 9$ جملة طباعة إجمالًا، نفس مبدأ العدّ وراء المجموع المزدوج $\sum_{r=1}^{3}\sum_{c=1}^{3}$.

## ⚠️ أخطاء شائعة

- **أخطاء الحدود مع `range`.** تتوقف `range(1, 10)` *قبل* 10، فتُعطي `1..9`. إن أردت تضمين 10، تحتاج `range(1, 11)`.
- **نسيان تحديث متغيّر الحلقة في `while`.** `while n > 0: print(n)` دون `n -= 1` بداخلها لا تتوقف أبدًا.
- **استخدام `=` بدلًا من `==` في شرط.** تلتقط بايثون هذا الخطأ فعليًا نيابة عنك (`if x = 5:` تُعتبر `SyntaxError`)، بخلاف بعض اللغات الأخرى — لكن يظل من المفيد معرفة لماذا سيكون هذا خطأً لو كان مسموحًا.
- **افتراض أن سلاسل `elif` تفحص كل الفروع.** بمجرد تطابق شرط `elif`/`if` واحد، يُتخطى الباقي تمامًا — الترتيب مهم، خصوصًا للشروط المتداخلة مثل حالة "القابلية للقسمة على 3 و5 معًا" في FizzBuzz.

## 🤔 أسئلة سقراطية

- ماذا يحدث إن كتبت `while True:` دون `break` بداخلها؟ جرّب ذلك بحذر في الملعب البرمجي (قد تحتاج لإيقاف التنفيذ يدويًا) — لماذا يختلف هذا عن حلقة `for` على `range` ثابت؟
- في FizzBuzz، لماذا يجب فحص حالة القابلية للقسمة على كليهما (`FizzBuzz`) *قبل* فحصي `Fizz`/`Buzz` المنفصلين، أو معالجتها بـ`and`؟ ما الذي سيحدث خطأً مع ترتيب `elif` ساذج؟
- تفعل `for i in range(5):` و`i = 0; while i < 5: ...; i += 1` نفس الشيء. في أي ظروف *ستضطر* لاستخدام `while` لأن `for` لا يستطيع التعبير عنها فعليًا؟
- يعني التقييم المختصر أن `a and b` تتخطى تقييم `b` إذا كانت `a` بالفعل `False`. هل يمكنك التفكير في موقف (بخلاف مثال القسمة على صفر أعلاه) يهم فيه هذا لأكثر من مجرد الأداء — حيث يكون تقييم `b` عندما لا ينبغي تقييمها خطأً فعليًا، وليس مجرد إهدار؟
- يُضمّن مثال جدول الضرب حلقة `for` داخل `for`. كيف سيبدو تضمين `while` داخل `for`، وهل يمكنك التفكير في مهمة حقيقية (وليس مجرد مثال مُختلق) يكون فيها هذا الدمج هو الخيار الطبيعي؟
